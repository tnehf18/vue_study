3. 양방향 바인딩(디렉티브) 

디렉티브

사용자가 입력한 엘리먼트 안의 내용에 따라 동적으로 변경. → 있어보이지만, 실제 얼마나 많은 곳에 활용할 수 있을지는 미지수.
> 실제로 많이 쓰더라, 그런데 컴포넌트 개념을 제대로 잡아야 많이 활용할 수 있다.

● v-show
  css 의 display 속성의 toggle 기능을 줄 뿐이며, 별로 중요하지 않은 기능이라선지 설명이 다들 다르다.
  인자로 any 으로 들어올 수 있는데 말 그대로 v-show의 인자로 받은 값이 존재하거나 true 이면 보이고,
  아니면 안 보이게 display 속성에 none 처리를 하는 게 전부이다. 

  인자로 boolean 을 직접적으로 줄 수도 있고, 특정 data 를 지정해 연동할 수 있다.

  번역체라서 설명이 굉장히 좋지 않고, 사라지는 대상이 왜 버튼인지 이해할 수 없다. 교재가 정말 안 좋다고 느껴진다.

● v-if
  단순 디렉티브로는 v-show 와 거의 차이가 없음. 그러나 if ~ else 기능 때문에, 단순히 display 속성의 토글 기능만 있는 show
  에 비해 더 입맛대로 만들 수 있다는 게 차이가 있음.

  또한, v-if 는 DOM 에 유지 되지 않는 차이가 존재.

  ※ <template> 태그, 해당 태그는 html5 에 최근에 등장한 태그로 꽤나 많은 브라우저에서 지원이 안된다.
     
     실제 페이지에 렌더링 되지 않기 때문, 따라서 vue, react 등 프론트 엔드에서 서버를 이용하는 모듈에서는 잘 활용되지만
     단순히 html 에 붙이기에는 브라우저 호환성 때문에 사용하기 어려울 듯하다. 

  v-show는 template 태그에 사용할 수 없단다. 
  (당연하게도 렌더링 되지 않고 동적으로 구현되는 태그이기 때문. 쉽게 말해 태그자체가 없어지므로 display 토글 못한다고 에러 난다)

● v-if v-else
  거의 당연한 내용이라 생략함.

  이걸로 다중 if 문 중첩하는 것은 안하는 게 더 나을 것 같음.
  억지로 하는 것도 상당히 별로인 것으로 보임.
  교재에는 아무런 언급이 없지만 v-else-if 가 존재한다.
